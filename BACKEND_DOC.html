📌 نظرة عامة

تطبيق NaqlGo مبني على ثلاث طبقات رئيسية، كل واحدة منها تخدم وظيفة محدّدة، وتتعاون فيما بينها لتقديم تجربة مستخدم سلسة:

🖥️ Server

يستقبل كل طلبات HTTP من التطبيق، يشغّل الكود، ويوزّع المهام.

🔧 Backend

المنطق الخلفي: المصادقة، الاستعلامات، إرسال الإشعارات، التتبع.

🐘 Database

يحفظ كل البيانات: المستخدمين، الطلبات، الرسائل، التقييمات.

المبدأ: الـ Server يستقبل الطلب، الـ Backend يقرّر ما يفعل، والـ Database يحفظ النتيجة. كل قطعة منفصلة لكنها تعمل في ميلي ثانية واحدة.

🖥️ 1. SERVER — الخادم

الخادم الرئيسي: Vercel

Vercel هو المنصة التي تستضيف التطبيق وتشغّل كل الـ APIs. إنه serverless، مما يعني أنك لا تدير خوادم بنفسك — Vercel يفعل ذلك تلقائياً.

🔧 2. BACKEND — المنطق الخلفي

ما هو الـ Backend؟

الـ Backend هو الكود الذي يعمل على Vercel. عندما يطلب المستخدم شيئاً من التطبيق (مثل تسجيل دخول أو إنشاء طلب)، الـ Backend هو الذي يقرّر ماذا يفعل: هل المستخدم مُصرَّح له؟ هل البيانات صحيحة؟ يحفظها في DB، يُرسل إشعار، ويرد على التطبيق.

🐘 3. DATABASE — قاعدة البيانات

التقنية: PostgreSQL على Neon

لماذا PostgreSQL؟

  • ✓ معيار صناعي عالمي — يستخدمه Apple, Netflix, Reddit, Instagram
  • ✓ قوي مع البيانات المعقدة (طلبات + عروض + رسائل + GPS مرتبطة)
  • ✓ مفتوح المصدر — لا يحبسك في شركة واحدة
  • ✓ موثوق منذ 1996 (30 سنة من التطوير)

لماذا Neon؟

  • ✓ Serverless — يدفع حسب الاستخدام، لا أوقات خمول مكلفة
  • ✓ HTTP-friendly — لا connection pool ينفجر
  • ✓ EU Frankfurt — قريب جغرافياً من الجزائر (~50ms)
  • ✓ Branches — يمكن إنشاء نسخة للاختبار بضغطة
  • ✓ Auto-backups — نسخ احتياطية تلقائية

الـ Indexes الـ 14

الـ Indexes هي ما يجعل الاستعلامات سريعة حتى مع آلاف المستخدمين. كل index هو "كتاب دليل" لجدول معيّن، يسمح لـ DB بالعثور على البيانات بسرعة O(log n). 

-- users (4 indexes)

idx_users_created           (createdAt DESC)
idx_users_kyc_pending       (kycReviewedAt, role)

سيناريو حقيقي: عميل يطلب توصيل

#الحدثالزمن

2APK يرسل POST إلى /api/requests~50 ms (شبكة)

4NextAuth: يتحقق من session ✓~5 ms

✅ الخلاصة

3 نقاط أساسية للتذكّر

  1. الخادم لا يحتاج إدارة: Vercel يدير كل شيء تلقائياً. لا "ssh"، لا تثبيت، لا تحديث نظام.
  2. قاعدة البيانات تكبر معك: Neon serverless، تدفع فقط على ما تستخدم، ينمو بسلاسة مع التطبيق.
  3. المنطق كله TypeScript قابل للقراءة: أي مطوّر جزائري يستطيع تعديل التطبيق بدون منحنى تعلّم.
